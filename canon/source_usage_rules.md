# Source Usage Rules

Этот файл задаёт порядок использования источников для академической предыстории.

## Главный принцип

Не всё в репозитории имеет одинаковый приоритет.

Перед сценой движок должен брать только нужные источники, а не тащить всё подряд.

## Приоритет источников

1. `gpt/engine_prompt.md`
2. `gpt/scene_format.md`
3. `canon/prequel_base.md`
4. `canon/public_lore.md`
5. `canon/hidden_observer_lore.md` — только если в сцене нужен Наблюдатель/Эйрен/ошибка реальности.
6. `canon/character_story_roles.md`
7. `canon/social_dynamics.md`
8. Файлы активных персонажей из `characters/main/`
9. Файлы скрытых персонажей из `characters/hidden/` — только если персонаж реально участвует или должен быть проверен.
10. `state/*.json` — только для текущего положения, отношений, знаний, репутации, инвентаря и истории сцены.

## Что не грузить всегда

Не грузить постоянно:

- всех персонажей;
- все скрытые файлы;
- весь hidden lore;
- все state-файлы;
- все правила академии;
- весь registry NPC.

Использовать точечно.

## Если сцена социальная

Подключить:

- `canon/social_dynamics.md`;
- активных персонажей;
- `state/relationships.json`;
- `state/reputation_state.json`;
- `state/knowledge_state.json`, если кто-то должен знать/не знать факт.

## Если сцена тренировочная

Подключить:

- `canon/academy_rating_and_training.md`;
- `canon/academy_energy_application_rules.md`;
- `canon/academy_combat_and_weapon_rules.md`;
- активных персонажей;
- `state/reputation_state.json`;
- `state/knowledge_state.json`.

## Если сцена с Наблюдателем

Подключить:

- `canon/hidden_observer_lore.md`;
- `characters/hidden/eiren_vale.md`, если есть;
- `state/knowledge_state.json`;
- `state/story_lines.json`.

Не раскрывать hidden lore другим персонажам автоматически.

## Если сцена с новым NPC

Если NPC важен или может повториться:

- добавить в `characters/npc/npc_registry.md`;
- если стал важным — создать отдельный файл;
- обновить `characters/character_id_index.md`.

Если NPC одноразовый:

- не создавать отдельный файл без необходимости;
- можно оставить в истории сцены.

## Если конфликт источников

Порядок решения:

1. Последний явный canon-файл.
2. Основной файл персонажа.
3. State-файлы текущей сессии.
4. Старые заметки/история.

Если противоречие критичное — не угадывать. Зафиксировать конфликт и спросить владельца канона.

## Запрещено

- Не придумывать hidden lore, если нет файла.
- Не переносить знания игрока в голову NPC.
- Не раскрывать правду о кайросах/цикле/Наблюдателе без сценического основания.
- Не использовать future knowledge как настоящее знание персонажей.
- Не грузить всё подряд при каждом ходе.
