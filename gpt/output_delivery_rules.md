# Output Delivery Rules

Этот файл регулирует порядок вывода игровой сцены и сохранения хода.

## Главное правило

Пользователь должен получить полноценную игровую сцену, а не техническое сообщение о сохранении.

Правильный порядок игрового хода:

1. Получить контекст через `getTurnContext`.
2. Написать полноценную сцену в формате `gpt/scene_format.md`.
3. Показать сцену пользователю как основной ответ.
4. Вызвать `submitTurnResult` или `applyTurnResult` для сохранения.
5. Не заменять сцену текстом результата сохранения.

## Запрещено

Не отвечать пользователю только так:

```txt
Ход успешно сохранён.
Акира протянула документы...
Можно выбрать следующий шаг.
```

Это техническое summary, а не сцена.

## Если сохранение прошло

Если сохранение прошло успешно, не надо показывать пользователю отдельную техническую фразу.

Достаточно уже показанной сцены.

## Если сохранение не прошло

Если сцена уже написана, но сохранение не прошло:

- не переписывать сцену в summary;
- коротко сообщить после сцены: `Технически: сцену не удалось сохранить, state не обновлён.`;
- не придумывать новый state.

## submitTurnResult

В `scene_text` можно сохранять краткое summary сцены, если полный текст с декоративными линиями, кавычками или переносами не проходит через JSON.

Но user-facing ответ всё равно должен оставаться полноценной сценой.

Краткое summary для сохранения не должно заменять сцену в чате.
